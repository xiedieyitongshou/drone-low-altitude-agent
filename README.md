# 低空作业气象决策 Agent

这是一个从比赛原型继续重构的项目。

最开始，这个项目是基于阿里云百炼工作流做出来的一个 Demo，场景是无人机巡检、低空巡航这类任务前的气象判断。原型已经跑通了从自然语言输入、天气查询、规则判断到结果输出的基本链路，但主要还是平台化流程，代码层面的工程化还比较弱。

现在这个仓库的目标，是把原来的原型整理成一个可以本地开发、持续迭代、后续能够部署和展示的项目。

## 这个项目想解决什么问题

这个项目不是单纯做天气查询，而是想回答一个更贴近业务的问题：

> 在某个地点、某个时间段内，当前天气条件下是否适合执行无人机巡检或低空巡航任务？

相比直接看天气数据，这个项目更关注：

- 能不能执行任务
- 风险主要来自哪里
- 哪个时间段更适合执行
- 多个地点里哪个更适合先去

## 这个项目是怎么来的

这个项目的上一版主要依赖百炼工作流和外部天气 API，整体思路大致是：

1. 用户输入一个自然语言问题  
   比如“明天上午 6 点到 9 点，深圳湾适合无人机巡检吗？”

2. 系统解析出结构化参数  
   包括地点、日期、时间段、任务类型

3. 查询逐小时天气和天气预警

4. 用固定规则做逐小时风险判断

5. 汇总整个时间段的结论  
   输出 `适飞 / 慎飞 / 禁飞`

6. 最后再生成自然语言说明

原型的核心价值在于：  
**关键判断不是交给大模型自由发挥，而是由规则来控制。**

## 计划实现的主要功能

这个项目后续希望逐步实现这些能力：

### 1. 单地点时间段评估

输入地点、日期、时间段、任务类型，输出：

- 每小时风险判断
- 整体结论
- 风险原因说明

### 2. 最佳执行时间推荐

如果当前时间段不适合执行任务，系统可以继续推荐：

- 更合适的执行窗口
- 连续风险更低的时间段

### 3. 多地点任务比选

如果同一任务有多个候选地点，系统可以做：

- 多地点风险对比
- 推荐优先执行地点

### 4. 历史任务复盘

后续希望保留每次查询的输入、天气快照和判断结果，方便：

- 回看某次任务为什么禁飞
- 做经验总结

### 5. 任务类型差异化规则

不同低空任务对天气的容忍度不一样，后续会把规则拆开，比如：

- 巡航
- 巡检
- 定点悬停拍摄
- 测绘

## 当前判断逻辑的基本思路

目前保留自上一版原型的核心逻辑是：

- 先按逐小时天气做判断
- 再结合天气预警做修正
- 最后按“最差小时原则”汇总整个时间段结果

也就是说：

- 只要某一个小时 `禁飞`，整个时间段就 `禁飞`
- 否则只要某一个小时 `慎飞`，整个时间段就 `慎飞`
- 所有小时都安全，才会判成 `适飞`

## 为什么要重构

原来的百炼版本适合快速验证思路，但不适合作为长期维护的项目。主要问题有：

- 逻辑过于依赖平台节点
- 代码结构不清晰
- 不方便测试和扩展
- 不方便做部署和 GitHub 展示

所以这次重构会把重点放在：

- 本地代码化
- 规则引擎拆分
- 接口服务化
- 后续的 Memory、RAG 和部署能力
